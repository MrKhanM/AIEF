# Purpose

This glossary defines the official terminology used throughout AIEF.

Every document, module, workflow, API, and codebase should use these definitions consistently.

---

# Core Concepts

## AIEF

Artificial Intelligence Engineering Framework.

The complete platform responsible for orchestrating AI systems through workflows, modules, memory, and intelligent execution.

---

## User

The individual or organization using AIEF.

---

## Workspace

A collection of projects owned by a user or team.

---

## Project

A container for related work.

Projects maintain their own workflows, files, modules, history, and memory.

---

## Workflow

A structured sequence of tasks designed to accomplish a user objective.

Example:

Build API

↓

Architecture

↓

Implementation

↓

Testing

↓

Documentation

---

## Task

The smallest unit of work executed inside a workflow.

---

## Module

A reusable expert responsible for a specific capability.

Examples:

- Product Manager
- Backend Architect
- SQL Expert
- UI Designer
- QA Engineer

Modules are platform-independent.

---

## AI Provider

An external AI service.

Examples:

- OpenAI
- Anthropic
- Gemini
- Mistral
- DeepSeek

Providers execute tasks assigned by modules.

---

## AI Router

The component responsible for selecting the most appropriate AI provider.

---

## Execution Plan

The complete strategy generated before a workflow begins.

---

## Review Cycle

A validation stage where outputs are checked, improved, and approved.

---

## Memory

Persistent knowledge stored across workflow executions.

Memory includes:

- Decisions
- Context
- Project History
- User Preferences

---

## Context

The information provided to a module before execution.

---

## Knowledge Graph

The structured relationship between projects, workflows, modules, memory, and outputs.

---

## Artifact

Any output generated by AIEF.

Examples:

- Source Code
- Documents
- Reports
- Diagrams
- Images
- Presentations

---

## Provider Adapter

The integration layer between AIEF and an AI provider.

Adapters make providers interchangeable.

---

## Capability

A measurable ability of an AI provider or module.

Examples:

- Code Generation
- Research
- Reasoning
- UI Design
- Planning

---

## Confidence Score

A numerical estimate of how reliable an output is.

---

## Consensus

Agreement reached after multiple providers review the same output.

---

## Workflow Engine

The core engine responsible for planning, executing, and managing workflows.

---

## Module Registry

The centralized catalog containing every available module.

---

## Session

A temporary interaction between the user and AIEF.

---

## Execution History

The complete record of workflow executions.
